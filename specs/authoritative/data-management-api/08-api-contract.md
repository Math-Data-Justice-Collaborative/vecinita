# Data Management API — API Contract

> Auto-generated: 2026-05-12

## Overview

The data-management API exposes a small REST surface: health, job proxy, embed,
and predict. The jobs proxy is a passthrough to the scraper service; embed and
predict use typed Pydantic schemas.

## Base URL

| Environment | URL |
|-------------|-----|
| Local dev | `http://localhost:8000` |
| Render | `https://vecinita-data-management-api-v1.onrender.com` |

## Endpoints

### GET /health

| Property | Value |
|----------|-------|
| Auth | none |
| Request body | — |
| Response (200) | `{ "status": "ok", "service": "vecinita-scraper" }` |
| Error responses | 503 (scraper unreachable or DB URL invalid) |
| Rate limit | none |

Validates `DATABASE_URL` via `corpus_db_guard`, then delegates to
`ScraperClient.health()`.

**Source:** `apis/data-management-api/apps/backend/vecinita_dm_api/routers/health.py`

### GET /jobs

| Property | Value |
|----------|-------|
| Auth | Bearer token (forwarded to scraper) |
| Query params | `user_id` (optional), `limit` (1–100, default 50) |
| Response (200) | `ScrapeJobListResponse { user_id, limit, jobs[], total }` |
| Error responses | 401 (missing auth), 403 (invalid key), 503 (scraper down) |
| Rate limit | none |
