# Pact provider verification (feature `005-wire-services-dm-front`)

Consumer pacts are generated by Vitest:

- Chat: `cd frontend && npm run test:pact` → `frontend/pacts/chat-frontend-vecinita-gateway.json`
- DM: `cd apps/data-management-frontend && npm run test:pact` → `apps/data-management-frontend/pacts/dm-frontend-*.json` (uses ``VITE_DM_API_BASE_URL`` toward the data-management API: ``GET /health``, auth probe ``GET /jobs?limit=1``, ``GET /documents``, ``GET/POST /jobs`` and ``GET /jobs/{id}``, ``POST /jobs/{id}/cancel``; consumer asserts the mock base is never ``*.modal.run`` — SC-001 / SC-005)
- Gateway→Agent: `pytest backend/tests/pact/test_gateway_agent_consumer_pact.py -q` → `backend/pacts/vecinita-gateway-vecinita-agent.json`
- Gateway→Modal SDK (sync message): `pytest backend/tests/pact/test_gateway_modal_sdk_message_pact.py -q` → `backend/pacts/vecinita-gateway-vecinita-modal-sdk.json`
- Gateway→Embedding HTTP (Modal RPC off, ``LOCAL_EMBEDDING_SERVICE_URL`` to a non-Modal service): `pytest backend/tests/pact/test_gateway_embedding_http_consumer_pact.py -q` → `backend/pacts/vecinita-gateway-vecinita-embedding-http.json`
- DM ``service_clients`` → Modal SDK (sync message, embedding/model/scraper health invokers): from repo root, ``cd backend && PYTHONPATH=../services/data-management-api/packages/service-clients:../services/data-management-api/packages/shared-schemas:../services/data-management-api/packages/shared-config:../services/data-management-api/apps/backend uv run pytest ../services/data-management-api/packages/service-clients/tests/pact/test_dm_service_clients_modal_message_pact.py -q`` → ``services/data-management-api/pacts/vecinita-dm-service-clients-vecinita-modal-sdk.json``
- DM API ``jobs_proxy`` → scraper ``/jobs`` HTTP (``ScraperClient.forward_jobs``): ``cd backend &&`` same ``PYTHONPATH`` as above ``uv run pytest ../services/data-management-api/tests/pact/dm_api_jobs_proxy_scraper_consumer_pact.py -q`` → ``services/data-management-api/pacts/vecinita-dm-api-vecinita-scraper-jobs-http.json``

Provider replay (requires **live** services matching the pact paths):

```bash
export PACT_PROVIDER_GATEWAY_URL=http://127.0.0.1:8004
pytest backend/tests/pact/test_chat_gateway_provider_verify.py -q

export PACT_PROVIDER_DM_API_URL=http://127.0.0.1:8005

export PACT_PROVIDER_AGENT_URL=http://127.0.0.1:8000

cd backend && PYTHONPATH=../services/data-management-api/packages/service-clients:../services/data-management-api/packages/shared-schemas:../services/data-management-api/packages/shared-config:../services/data-management-api/apps/backend uv run pytest ../services/data-management-api/packages/service-clients/tests/pact/test_dm_service_clients_modal_message_pact_provider_verify.py -q
```

Root **Makefile** target **`pact-verify-providers`** runs the HTTP provider checks plus the Modal SDK message verifiers (they **skip** unless the matching env var is set and the generated pact JSON exists).

## CI / dependency parity

Python consumer modules under ``tests/pact/`` import ``pact`` at **module import** time. GitHub Actions and ``make ci`` install dependencies with ``uv sync --frozen --extra ci``, which resolves **`vecinita[ci]`** only; UV ``[dependency-groups]`` are a separate knob. If ``pact-python`` were listed only in a dependency group and not in the **`ci``** extra, pytest would fail during **collection** with ``ModuleNotFoundError: No module named 'pact'`` (markers are applied after imports). See **TESTING_DOCUMENTATION.md** (“Backend `vecinita[ci]` …”) and ``tests/contracts/test_backend_ci_extra_includes_pact_python_contract.py``.
