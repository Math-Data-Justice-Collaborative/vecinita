## Overview

Data flows are build-time only. Markdown content is read from the monorepo, transformed into static HTML, and served to users.

## Inbound Data

| Source | Format | Trigger | Destination |
|--------|--------|---------|-------------|
| `docs/` directory | Markdown (.md) files | Build command | Docusaurus build pipeline |
| `docusaurus.config.ts` | TypeScript config | Build command | Docusaurus configuration |
| `sidebars.ts` | TypeScript config | Build command | Sidebar auto-generation |
| `src/css/custom.css` | CSS | Build command | Theme customization |
| `static/img/logo.svg` | SVG | Build command | Site favicon/logo |

## Internal Processing

| Stage | Input | Transformation | Output |
|-------|-------|----------------|--------|
| Content reading | Markdown files from `../docs/` | Docusaurus reads and parses | MDX AST |
| Sidebar generation | Directory structure | `autogenerated` sidebar config | Navigation tree |
| MDX rendering | MDX AST | React component rendering | HTML pages |
| Asset optimization | CSS, JS, images | Webpack bundling + minification | Optimized static assets |
| Link validation | All internal links | Broken link checker | Warnings (configured as `warn`) |

## Outbound Data

| Destination | Format | Trigger | Content |
|-------------|--------|---------|---------|
| `build/` directory | Static HTML/CSS/JS | `npm run build` | Complete static site |
| Browser | HTTP responses | User request | Static pages |

## Data Persistence
